GSC Exporter: Weekly and Monthly Reporting Blueprint

This blueprint outlines the recommended scheduling, report selection, and workflow to establish a comprehensive search analytics pipeline for your properties.

Core Pipeline Objective

A successful search monitoring pipeline balances tactical agility (weekly monitoring to spot technical drops or content anomalies) with strategic analysis (monthly reports to evaluate long-term trends, keyword cannibalisation, and new search features).

1. Weekly Reporting: Monitoring & Agility

Weekly reporting focuses on operational monitoring: identifying keyword trends, tracking recent content launches, and checking search features like Google Discover and Google News for editorial and news properties.

Report Operational Purpose Frequency & Timing
Canary Monitoring
Weekly 		Provides a multi-period delta analysis (LW, LM, LY) across clicks, impressions, CTR, and average position for all key properties. Good for flagging sudden shifts. Run every Monday morning to review the preceding week's performance.
Period Comparison
Weekly 		Compares the last 7 days to the previous 7 days. Highlights keyword/page query deltas and displays interactive performance charts. Run weekly to evaluate the impact of content edits, recent launches, or technical adjustments.
Page-Level Performance
Weekly 		Analyses individual URL metrics across search types (Web, Discover, News) to show which URLs drove the week's traffic. Run weekly for properties that receive substantial non-standard search traffic.

Weekly Execution Commands

Use the --last-7-days flag. The suite automatically determines the latest available date in GSC (accounting for the standard 2 to 3 day processing lag) and anchors the 7-day reporting range accordingly.

Canary Monitoring Report

Checks all properties configured in your config/properties.json file:

python reports/monitoring/canary_report.py --config config/properties.json
Period Comparison Report

Compares the last 7 days against the previous 7 days for a specific domain:

python reports/period_comparison_report.py sc-domain:example.com --last-7-days
Page-Level Performance (by Search Type)

Run separately for standard Web search, Google Discover, and Google News:

python reports/page_level_report.py sc-domain:example.com --search-type web --last-7-days
python reports/page_level_report.py sc-domain:example.com --search-type discover --last-7-days
python reports/page_level_report.py sc-domain:example.com --search-type news --last-7-days

2. Monthly Reporting: Strategic Trend Analysis

Monthly reporting provides a deeper, strategic audit: evaluating search visibility changes, keyword cannibalisation, ranking distributions, and identifying content optimization opportunities.

Report Strategic Purpose Frequency & Timing
Performance Analysis
Monthly 		Categorises pages into actionable cohorts: "Rising Stars" (gaining visibility), "Content Decay" (losing clicks), and "Low CTR Opportunities" (high impressions, low CTR). Run at the start of the month to set the content optimisation priorities for the next 30 days.
GSC Pages & Queries
Monthly 		Provides an interactive drill-down report pairing top pages with their driving search queries, including brand vs non-brand performance. Run monthly to audit keyword relevance and user intent mapping.
Keyword Cannibalisation
Monthly 		Flags queries where multiple pages compete in the search results, diluting rankings and CTR. Run monthly to identify where content consolidation or internal linking changes are needed.
Query Position Analysis
Monthly 		Tracks keyword ranking distribution (Positions 1-3, 4-10, 11-20, etc.) over a 16-month period to identify long-term structural changes. Run monthly or quarterly to monitor overall brand search visibility trends.
Monthly Summary
Monthly 		Generates a clean, aggregate executive overview of the calendar month's totals. Run monthly to archive high-level performance metrics for historical logging.

Monthly Execution: The Warmed Cache Workflow

To avoid hitting Google Search Console API rate limits and to make report generation near-instant, always run the Cache Warmer first. The warmer pre-fetches and saves the "Golden Caches" locally.

1 Warm the Cache

Prime the cache for your sites (this pulls the necessary monthly files for standard and granular reporting):

python utilities/cache_warmer.py --file site-lists/sites.txt
2 Run Individual Strategic Reports

Once the cache is primed, execute the primary monthly analysis tools using the --last-month flag (which automatically resolves to the last complete calendar month):

Performance Analysis (Optimisation Opportunities)
python reports/performance_analysis.py sc-domain:example.com --last-month
GSC Pages & Queries Drill-down
python reports/gsc_pages_queries.py sc-domain:example.com --last-month
Keyword Cannibalisation Auditor
python reports/keyword_cannibalisation_report.py sc-domain:example.com --last-month
Query Position Analysis (16-Month Trend)
python reports/query_position_analysis.py sc-domain:example.com --last-month
Monthly Summary Report
python reports/monthly_summary_report.py sc-domain:example.com --last-month
3 Alternative: Run the Automated Site Suite

To run all primary analysis reports for a single domain in a single command, use the suite runner:

python run_all_reports_for_site.py sc-domain:example.com --last-month

3. Corporate Presentation & Stakeholder Communication

Running the code is only half the task; translating the output into business value is key to promoting your work within the organisation. Configure your reports for distinct stakeholder groups:

For Content & Editorial Teams
Focus: Immediate Performance

Editorial teams care about which articles worked. Send them the weekly Page-Level Performance report (split by Web, Discover, and News). Highlight high Discover click counts, as it reinforces headline quality and topic virality.

For SEO & Growth Managers
Focus: Actionable Optimisation

SEO specialists need optimization queues. Provide them with the monthly Performance Analysis (specifically the *High Impressions, Low CTR Opportunities* list) and the Keyword Cannibalisation audit to guide their remediation sprints.

For Executive Stakeholders
Focus: Macro Trends & Health

Directors and VPs want to see high-level trends without getting bogged down in individual search queries. Present them with the monthly Query Position Distribution trend charts and the Monthly Summary report.

For Annual Reviews & Wrap-ups
Focus: Performance Celebrations

Use the GSC Wrapped report at the end of the year or major quarters. It generates a visually engaging, card-based summary of top-performing assets, which fits well into corporate newsletters or review slide decks.

Subtle Corporate Promotion

To promote your initiative internally, you can apply subtle corporate branding to the generated HTML reports:

  • Customise the Footer: Replace the open-source GitHub link with an internal signature, e.g. Report generated by the Enterprise Search Analytics Suite (maintained by [Your Name/Team]).
  • Header Badge: Add an inline Bootstrap pill badge next to report titles (e.g., <span class="badge bg-secondary">Internal SEO Tool</span>) to mark it as an official internal resource.
  • Apply Corporate Colour Accents: Adjust the CSS tables header background-color from the default dark grey (#434343) to match your corporation's official primary brand colour.
