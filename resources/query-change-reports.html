Identifying Query Changes Week-on-Week

To get a quick view of significant query changes week-on-week, you can utilise the following reports.

1. Seasonal Query Spike Report (seasonal_query_spike_report.py)

This report is excellent for quickly identifying queries that are performing "out of the ordinary" compared to their historical averages. While designed for seasonal analysis, it can be adapted to highlight significant weekly changes.

How it works: The report calculates a Z-score for clicks and impressions for each query within a specified period, comparing it against the query's historical performance (defaulting to the last 16 months of data). Queries exceeding a defined threshold are flagged as "spikes."

For a quick view of weekly changes (anomalies):

  1. Specify a weekly date range: Run the report for the week you are interested in (e.g., last 7 days). The report will then compare that week's performance for each query against its longer-term historical average. This is not a direct "last week vs. this week" comparison, but rather "this week vs. historical norm for this query," which is often more useful for identifying unexpected changes.

    Example Command (Last 7 days):

    python seasonal_query_spike_report.py your-site-url.com --last-7-days --threshold 2.0 --min-clicks 10
    • Replace your-site-url.com with your actual Google Search Console property.
    • --last-7-days: Analyses data for the last full 7 days.
    • --threshold 2.0: Queries with a Z-score above 2.0 (meaning they are 2 standard deviations above the mean) will be flagged. You can adjust this value.
    • --min-clicks 10: Only queries with at least 10 clicks in the analysis period will be considered.
  2. Review the HTML Report: The generated HTML report will list queries that show significant spikes in clicks or impressions, allowing you to quickly spot anomalies.

2. Queries and Pages Analysis Report (queries_pages_analysis.py)

This report provides a detailed breakdown of clicks, impressions, CTR, and average position for each query and page combination within a given date range. This is useful for a more direct week-on-week comparison, although it requires manual review.

For a detailed week-on-week comparison:

  1. Run for Current Week: Generate a report for the most recent full week.

    Example Command (Current Week):

    python queries_pages_analysis.py your-site-url.com --start-date YYYY-MM-DD --end-date YYYY-MM-DD
  2. Run for Previous Week: Generate a report for the week immediately preceding the current week.

    Example Command (Previous Week):

    python queries_pages_analysis.py your-site-url.com --start-date YYYY-MM-DD --end-date YYYY-MM-DD
  3. Manually Compare Reports:
    • CSV Files: Open both weeks' CSV files in a spreadsheet program (e.g., Google Sheets, Excel). You can then use features like conditional formatting or pivot tables to highlight changes in metrics for individual queries.
    • HTML Reports: Review the HTML reports side-by-side to visually compare the top queries and their performance.

Recommendation for "Quick View":

For a "quick view" of what's changed (i.e., identifying unexpected increases or decreases), the Seasonal Query Spike Report is generally more effective as it automates the detection of anomalies. Use the Queries and Pages Analysis Report when you need to perform a precise, detailed comparison between two specific weekly periods.
