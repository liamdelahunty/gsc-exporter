Caching Recommendations

GSC Exporter: Hash-CSV + JSON Meta

Approach: Use CSV for tabular data (performance) and JSON for metadata.

  • Data: Store in {hash}.csv for fast Pandas access.
  • Metadata: Store in {hash}.json to track site, dates, and dimensions.
  • Benefit: Fast, compact, and allows multiple reports to share the same data by generating the same hash.

GA4 API Explorer: Pure JSON

Approach: Stick with the current hash-based JSON system.

  • Reason: Variable report structures and high metadata requirements make JSON ideal.
  • Benefit: Already robustly implemented in run_report.py.

Best Implementation Prompt for GSC

Implement a hash-based CSV caching system for GSC Exporter.
1. Create cache_manager.py to generate MD5 hashes from request params.
2. Save data to {hash}.csv and metadata to {hash}.json.
3. Update reports to check for these files before calling the API.
