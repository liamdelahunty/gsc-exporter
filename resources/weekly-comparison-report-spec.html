New Report Specification: Weekly Comparison

This document details the requirements for a new modular report, weekly_comparison_report.py, to be implemented in the reports/ directory.

1. Purpose

To provide a comparative performance analysis of two identical-length time periods (e.g., last 7 days vs. previous 7 days) for both top keywords and top URLs.

2. CLI Arguments

The report must support flexible date definition, prioritising ease of use for comparisons:

python reports/weekly_comparison_report.py site.com [args]
  • --period1-start YYYY-MM-DD / --period1-end YYYY-MM-DD
  • --period2-start YYYY-MM-DD / --period2-end YYYY-MM-DD
  • Convenience flag: --last-7-days. If used, period1 is automatically set to the last 7 days, and period2 is set to the 7 days preceding that.

3. Functionality

  1. Data Fetching:
    • Fetch GSC data for period1 (keywords and pages).
    • Fetch GSC data for period2 (keywords and pages).
  2. Analysis:
    • Join the two datasets on 'query' and 'page' respectively.
    • Calculate the delta (absolute and percentage) for clicks, impressions, and CTR for each entity.
  3. Top Entities:
    • Sort results by the difference in clicks.
    • Extract the Top 250 keywords.
    • Extract the Top 250 URLs.

4. Output Requirements

  • CSV: One file containing comparative metrics for both Top 250 keywords and Top 250 URLs.
  • HTML: A comparative dashboard displaying two tables: "Top 250 Keyword Changes" and "Top 250 URL Changes", highlighting positive and negative changes in clicks/impressions/CTR.
